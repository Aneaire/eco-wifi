EcoWiFi Webserver Guide

Build Your EcoWiFi Webserver

Follow this comprehensive guide to set up the backend webserver that powers your EcoWiFi system, handling bottle detection, user sessions, and WiFi access management.

1

Prerequisites

System Requirements:

  • Linux-based system (Ubuntu/Armbian)
  • 1GB RAM minimum
  • 8GB storage space
  • Network connectivity

Software Dependencies:

sudo apt update
sudo apt install curl git build-essential

Tip: Ensure your system is up to date and has internet access for package installation.

2

Bun Runtime Setup

Install Bun:

curl -fsSL https://bun.sh/install | bash
source ~/.bashrc

Verify Installation:

bun --version

Note: Bun is a fast JavaScript runtime that replaces Node.js for this project.

3

Project Initialization

Create Project Directory:

mkdir ecowifi-server
cd ecowifi-server

Initialize with Bun:

bun init

Install Dependencies:

bun add hono better-sqlite3 @hono/node-server
4

Database Configuration

Database Schema:

users: id, mac_address, session_start, credits

bottle_logs: id, user_id, timestamp, material_type

system_stats: id, timestamp, total_bottles, active_sessions

Migration Script:

bun run src/migrate.ts

Info: The database uses SQLite with better-sqlite3 for optimal performance.

5

API Routes Setup

Core Endpoints:

  • • POST /api/bottle - Log bottle insertion
  • • GET /api/session - Check user session
  • • POST /api/auth - Authenticate user
  • • GET /api/stats - System statistics

Route Structure:

src/routes/
├── bottle.ts
├── auth.ts
├── stats.ts
└── index.ts

Production Deployment

Build Process:

bun build src/index.ts --outdir ./dist --target bun

Start Server:

bun run dist/index.js
